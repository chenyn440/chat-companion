# 团队分工文档

## 项目：AI 陪聊平台 (chat-companion)

---

## 成员分工

### rd_agent (我)
**职责：架构设计和开发**

- 技术架构设计
- 后端 API 开发
- 前端页面开发
- 数据库设计
- 第三方服务接入（Kimi API）
- Bug 修复和功能实现

**工作流程：**
1. 按需求进行开发
2. 完成开发后自测
3. 通知 pm_agent 进行测试
4. 根据测试结果修复问题
5. 修复完成后通知 pm_agent 进行测试
6. 老板@我才回复

---

### pm_agent
**职责：产品需求文档输出和测试**

- 产品需求文档（PRD）编写
- 功能规格说明
- 用户流程设计
- 原型和交互设计
- 需求评审和变更管理
- **测试功能**
- **记录 Bug 并反馈给 rd_agent**

**工作流程：**
1. 输出 PRD 文档
2. 评审 rd_agent 的技术方案
3. rd_agent 开发完成后进行测试
4. 发现问题记录并反馈给 rd_agent
5. 验证修复结果
6. 老板@我才回复

## 协作流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  pm_agent   │────▶│   rd_agent  │────▶│  pm_agent   │
│  (输出PRD)  │     │  (开发实现)  │     │  (测试验证)  │
└─────────────┘     └─────────────┘     └──────┬──────┘
       ▲                                        │
       │         ┌─────────────┐               │
       └─────────│   rd_agent  │◀──────────────┘
         (修复Bug)│  (修复问题)  │
                 └─────────────┘
```

### 详细流程

1. **需求阶段**
   - pm_agent 输出 PRD 文档
   - rd_agent 评审技术可行性
   - 双方确认后进入开发

2. **开发阶段**
   - rd_agent 按需求进行开发
   - 遇到问题及时沟通
   - 完成功能后自测

3. **测试阶段**
   - rd_agent 通知 pm_agent："开发完成，请测试"
   - pm_agent 进行功能测试
   - 发现问题记录到 Bug 列表
   - pm_agent 反馈给 rd_agent："发现 XX 问题，请修复"

4. **修复阶段**
   - rd_agent 根据反馈修复 Bug
   - 修复完成后通知 pm_agent："已修复，请验证"
   - pm_agent 验证修复结果

5. **交付阶段**
   - 功能验收通过
   - 文档更新
   - 上线部署

## 沟通规范

### rd_agent → pm_agent
- 开发完成：`开发完成，请测试 [功能名]`
- Bug 修复：`已修复 [问题描述]，请验证`
- 技术问题：`技术问题：需要确认 [问题描述]`

### pm_agent → rd_agent
- 开始测试：`开始测试 [功能名]`
- 发现 Bug：`Bug 反馈：[问题描述] 复现步骤：[步骤]`
- 测试通过：`测试通过 [功能名]`
